# Tourgether 피드 기반 공유경제 플로우

**작성일**: 2025년 12월 4일

---

## 핵심 원칙

> **모든 것은 피드에서 시작된다**

Tourgether는 SNS입니다. 상품을 판매하는 이커머스가 아닙니다.
- 여행 경험을 공유하는 피드
- 서비스를 제안하는 피드
- 여행 요구사항을 요청하는 피드

모든 경제 활동은 피드 → DM → 계약 → 정산 순서로 진행됩니다.

---

## 1. 사용자 역할

### 양방향 역할 시스템

| 역할 | 설명 | 활동 |
|------|------|------|
| **여행자** | 여행을 계획하거나 진행 중인 사용자 | 요구사항 등록, 서비스 요청, 경험 공유 |
| **로컬가이드** | 서비스를 제공하는 사용자 | 플랜 제안, 서비스 제공, 정산 수령 |

**핵심:** 누구나 여행자이면서 로컬가이드가 될 수 있음!
- 서울 여행 중인 A는 → 여행자
- 서울에서 맛집 투어를 제공하는 A는 → 로컬가이드
- 같은 사람이 상황에 따라 역할 전환

## 2. 피드 유형

### 2.1 여행 포스트 (SNS 핵심)

```
[일반 여행 포스트]
├── 사진/영상
├── 위치 태그
├── 여행 스토리
├── 좋아요/댓글
└── 공유
```

목적: 체류 시간 증가, 커뮤니티 형성

### 2.2 여행자 요구사항 피드

```
[요구사항 포스트]
├── "이런 여행 하고싶어요"
├── 여행 일정/목적지
├── 예산 범위
├── 원하는 서비스 유형
└── 로컬가이드 제안 대기
```

예시:
- "12월 20-25일 도쿄 여행, 현지인만 아는 라멘 맛집 투어 원해요 (예산 10만원)"
- "발리에서 서핑 배우고 싶어요, 한국어 가능한 강사 찾습니다"

### 2.3 로컬가이드 서비스 제안 피드

```
[서비스 제안 포스트]
├── 제공 가능한 서비스
├── 지역/언어
├── 가격대
├── 포트폴리오 (이전 리뷰)
└── 즉시 문의 버튼
```

예시:
- "도쿄 현지인이 알려주는 히든 라멘 투어 (3시간, 5만원)"
- "발리 서핑 레슨 + 현지 가이드 (하루 15만원)"

## 4. 서비스 카테고리

### 온라인 서비스

| 카테고리 | 예시 | 가격대 |
|----------|------|--------|
| 예약대행 | 레스토랑 예약, 티켓 구매 | ₩5,000~₩20,000 |
| 정보제공 | 현지 맛집 리스트, 여행 팁 | ₩3,000~₩15,000 |
| 통역/번역 | 영상통화 통역, 문서 번역 | ₩10,000~₩50,000 |
| 일정 컨설팅 | 맞춤 여행 플랜 작성 | ₩20,000~₩100,000 |

### 오프라인 서비스

| 카테고리 | 예시 | 가격대 |
|----------|------|--------|
| 가이드 투어 | 맛집 투어, 시티 투어 | ₩30,000~₩150,000 |
| 차량 서비스 | 공항 픽업, 일일 투어 | ₩50,000~₩300,000 |
| 액티비티 | 서핑 레슨, 쿠킹 클래스 | ₩50,000~₩200,000 |
| 쇼핑대행 | 현지 쇼핑 동행 | ₩30,000~₩100,000 |

### 구매대행 (보따리)

| 카테고리 | 예시 | 수수료 |
|----------|------|--------|
| 셀프 보따리 | 여행자가 귀국 시 물건 사다주기 | 물건값의 10-20% |
| 현지 구매대행 | 로컬가이드가 물건 구매+배송 | 물건값의 15-25% |

## 5. 에스크로 결제 구조

### 5.1 마일스톤 결제

```
[간단한 서비스]
└── 전액 선결제 → 서비스 완료 → 전액 릴리스

[복잡한 서비스]
├── 계약금 30% → 계약 체결 시
├── 중도금 40% → 서비스 시작 시
└── 잔금 30% → 서비스 완료 시
```

### 5.2 에스크로 생명주기

```
CREATED → FUNDED → IN_PROGRESS → COMPLETED → RELEASED
                        ↓
                    DISPUTED → RESOLVED
                        ↓
                    REFUNDED (부분/전체)
```

### 5.3 분쟁 해결

| 상황 | 처리 |
|------|------|
| 서비스 미제공 | 72시간 후 자동 전액 환불 |
| 불만족 | 양측 협의 → 관리자 중재 |
| 노쇼 (여행자) | 취소 정책에 따라 처리 |
| 노쇼 (가이드) | 전액 환불 + 페널티 |

## 6. 데이터 모델 개요

### 피드 기반 엔티티

```
[posts] ─────────────────── 일반 여행 포스트
    │
    ├── [travel_requests] ─ 여행자 요구사항 (피드 확장)
    │       │
    │       └── [proposals] ─ 로컬가이드 제안 응답
    │
    └── [service_offers] ── 로컬가이드 서비스 제안 (피드 확장)
```

### 계약/거래 엔티티

```
[contracts] ───── 양자 합의 계약서
    │
    ├── [escrow_transactions] ─ 에스크로 거래
    │       │
    │       └── [milestones] ─ 마일스톤 결제
    │
    └── [disputes] ─ 분쟁 기록
```

## 7. 용어 정리

| 이전 (이커머스) | 현재 (SNS 공유경제) |
|----------------|---------------------|
| Experience (상품) | 서비스 제안 / 플랜 |
| Booking (구매) | 계약 (Contract) |
| Payment (결제) | 에스크로 거래 |
| Host (판매자) | 로컬가이드 |
| Guest (구매자) | 여행자 |
| Order (주문) | 계약 요청 |
| Product (상품) | 서비스 |

### SNS 지표 (체류)

| 지표 | 목표 |
|------|------|
| DAU/MAU | 30%+ |

| 포스트 작성률 | 5%+ |

| 제안 → 계약 전환율 | 15%+ |
| 계약 완료율 | 90%+ |
| 평균 거래 금액 | ₩50,000+ |
