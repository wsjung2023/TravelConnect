## 🔍 구조 이해 (필독)

### 맵 페이지 레이어 구조

```
AppShell (min-h-screen, pb-[72px])
  └─ Home page (height: calc(100vh - 72px), overflow: hidden)
       ├─ div.absolute.inset-0
       │    └─ MapComponent (h-full w-full)
       │         ├─ 지도 영역 (div ref=mapRef)
       │         ├─ [B1] 피드작성 FAB     (absolute bottom-24 right-4 z-10)  ← 문제
       │         ├─ 모드 토글 버튼        (absolute bottom-20 right-4 z-10)
       │         ├─ [B2] 주변 피드/체험 패널 (absolute bottom-0, z-index 없음) ← 문제
       │         └─ MiniPlanButton        (fixed bottom-24 right-4 z-[1000])
       │
       ├─ MapTopBar (floating)
       └─ [B3/B4] MapBottomSheet (absolute bottom-0 z-20)  ← 문제
```

### 현재 z-index 계층

| 요소 | z-index | 위치 방식 |
|------|---------|----------|
| AppBottomNav | z-30 | fixed bottom-0 |
| MiniPlanButton | z-[1000] | fixed bottom-24 |
| MapBottomSheet | z-20 | absolute bottom-0 (home div 기준) |
| 모드 토글 버튼 | z-10 | absolute bottom-20 (MapComponent 기준) |
| 피드작성 FAB | z-10 | absolute bottom-24 (MapComponent 기준) |
| 주변 피드/체험 패널 | 없음(0) | absolute bottom-0 (MapComponent 기준) |

### 핵심 충돌 계산 (home div 기준 bottom 기준점)

```
MapBottomSheet collapsed 상태: 0 ~ 132px 점유
피드작성 FAB: bottom-24(96px) ~ bottom-24+h-12(144px) 점유
겹치는 구간: 96 ~ 132px (36px) → FAB 48px 중 36px가 시트 뒤에 숨음
z-10 < z-20 이므로 시트가 FAB 위에 그려짐 → FAB 반쪽만 보임
```

### MapBottomSheet 스냅 상태

| 상태 | 높이 | 설명 |
|------|------|------|
| collapsed | 132px | 항상 보이는 기본 상태 |
| half | 54vh | 반쯤 열림 |
| expanded | 90vh | 완전히 펼침 |

  const dy = (e.changedTouches[0]?.clientY ?? 0) - touchStartY.current;
  const idx = SNAPS.indexOf(state);
  if (dy < -50 && idx < SNAPS.length - 1) onStateChange(SNAPS[idx + 1]!);

const onTouchEnd = (e: React.TouchEvent) => {
  if (touchStartY.current === null) return;
  const dy = (e.changedTouches[0]?.clientY ?? 0) - touchStartY.current;
  const idx = SNAPS.indexOf(state);
  if (dy < -50 && idx < SNAPS.length - 1) {
    onStateChange(SNAPS[idx + 1]!);

    // 빠른 다운 스와이프 → 바로 collapsed
    onStateChange('collapsed');
  } else if (dy > 50 && idx > 0) {
    onStateChange(SNAPS[idx - 1]!);

  touchStartY.current = null;
};

**위치**: 약 66~70번 줄

## ⚠️ 주의사항

### 1. absolute vs fixed 혼동 금지
- MapComponent 내부의 `absolute` 요소들은 **MapComponent 부모(home div)** 를 기준으로 위치 계산됨
- `MiniPlanButton`의 `fixed`는 **뷰포트** 기준
- 둘을 혼동하면 위치 계산이 전혀 달라짐

### 2. Tailwind 동적 클래스 (`z-[25]`, `z-[35]`, `bottom-[104px]`)
- Tailwind JIT가 대괄호 동적 값을 인식하는지 반드시 확인
- 만약 스타일이 적용 안 되면 `style={{ zIndex: 25 }}` 등 인라인 스타일로 대체

### 3. MapComponent는 2161줄의 대형 파일
- 수정 전 줄 번호 재확인 필수 (다른 세션에서의 변경으로 줄 번호가 달라질 수 있음)
- 파일 전체를 읽지 말고 `grep` 또는 `offset/limit`으로 해당 부분만 읽을 것

### 4. 기능·데이터 로직 변경 절대 금지
- MapBottomSheet의 추천 카드 내용 변경 없음
- 주변 피드 패널의 API 호출 / nearbyItems 데이터 로직 변경 없음
- 탭 항목(전체/피드/체험/열린 사람들) 수정 없음
- 이 작업은 **위치, z-index, 초기값, UX 인터랙션만** 수정

### 5. 수정 후 반드시 서버 재시작 확인
- 모든 수정은 프론트엔드 TSX 파일이므로 서버 재시작 없이 Vite HMR로 즉시 반영
- 하지만 변경 완료 후 서버 재기동 → 콘솔 에러 없는지 확인

### 6. 수정 범위를 지킬 것
- 가드레일: 허용된 파일 외 수정 금지
- 형식화(formatting), 리팩토링 등 요청 외 수정 금지
