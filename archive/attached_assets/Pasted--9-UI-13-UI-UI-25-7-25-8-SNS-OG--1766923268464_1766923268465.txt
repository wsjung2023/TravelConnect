1) 북마크(저장) 기능: 데이터 모델 + API
DB 테이블 (Supabase/Postgres 예시)
create table bookmarks (
  id uuid primary key default gen_random_uuid(),
  user_id uuid not null,
  post_id uuid not null,
  created_at timestamptz default now(),
  unique (user_id, post_id)
);

create index on bookmarks (user_id, created_at desc);
create index on bookmarks (post_id);


Firebase/Firestore라면:

bookmarks/{userId}/items/{postId} 형태로 만들어도 됨.

API (Next.js route 예시)

POST /api/bookmarks : 토글

GET /api/bookmarks?userId=... : 북마크 목록

// POST /api/bookmarks
// body: { postId: string }
export async function POST(req: Request) {
  const { postId } = await req.json();
  const userId = await getUserIdFromSession(); // 너네 auth 방식에 맞게

  // 존재하면 삭제, 없으면 생성 (toggle)
  const existing = await db.bookmark.findFirst({ where: { userId, postId }});
  if (existing) {
    await db.bookmark.delete({ where: { id: existing.id }});
    return Response.json({ bookmarked: false });
  } else {
    await db.bookmark.create({ data: { userId, postId }});
    return Response.json({ bookmarked: true });
  }
}

2) 피드 화면: 북마크 버튼 + 툴팁(#9)
UI 포인트

아이콘만 덜렁 두지 말고,

호버/롱프레스 시 “저장”/“저장됨” 툴팁 뜨게 마감.

Radix UI 쓰면 제일 깔끔함(툴팁 구현 빨라).

import * as Tooltip from "@radix-ui/react-tooltip";

function BookmarkButton({ postId, initialBookmarked }: { postId: string; initialBookmarked: boolean }) {
  const [bookmarked, setBookmarked] = React.useState(initialBookmarked);

  const toggle = async () => {
    const res = await fetch("/api/bookmarks", {
      method: "POST",
      headers: { "Content-Type": "application/json" },
      body: JSON.stringify({ postId }),
    });
    const data = await res.json();
    setBookmarked(data.bookmarked);
  };

  return (
    <Tooltip.Provider>
      <Tooltip.Root>
        <Tooltip.Trigger asChild>
          <button onClick={toggle} aria-label={bookmarked ? "저장됨" : "저장"}>
            {bookmarked ? "🔖" : "📑"}
          </button>
        </Tooltip.Trigger>
        <Tooltip.Content side="top" className="rounded px-2 py-1 text-sm shadow">
          {bookmarked ? "저장됨" : "북마크 저장"}
        </Tooltip.Content>
      </Tooltip.Root>
    </Tooltip.Provider>
  );
}

4) 피드 정렬 UI(#13) — “로직은 있는데 UI가 없음” 해결

여기서부터가 “SNS답게 보이느냐”를 결정해.
정렬 UI는 “작아도 있어야” 제품이 제품처럼 보임.

추천 정렬 옵션(현실적으로 MVP에서 쓸만한 것만)

최신순(latest)

인기순(popular) : 좋아요/북마크 수 기반

가까운순(nearby) : 현재 위치 기반 (옵션)

UI는 드롭다운이 제일 무난
function FeedSort({ value, onChange }: { value: string; onChange: (v:string)=>void }) {
  return (
    <select value={value} onChange={(e)=>onChange(e.target.value)}>
      <option value="latest">최신순</option>
      <option value="popular">인기순</option>
      <option value="nearby">가까운순</option>
    </select>
  );
}

정렬에 따라 쿼리 변경(서버에서 처리 추천)

GET /api/feed?sort=latest|popular|nearby

popular 구현(제일 현실적인 방식)

posts 테이블에 like_count, bookmark_count 컬럼을 캐싱해두고 정렬

요청마다 join해서 count 세면 느려짐

A) 가장 빠른 공유: Web Share API (모바일에서 강력)
async function sharePost({ url, title, text }: {url:string; title:string; text:string}) {
  if (navigator.share) {
    await navigator.share({ title, text, url });
    return;
  }
  await navigator.clipboard.writeText(url);
  alert("링크가 복사됐어요!");
}


이것만으로도:

안드로이드/아이폰: 카톡/인스타/문자/메일 “공유 패널” 자동 뜸
즉, “카카오톡 공유 기능 없음” 대부분은 이걸로 해결돼.
